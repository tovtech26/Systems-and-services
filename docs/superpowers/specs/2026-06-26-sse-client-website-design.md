# SSE Client Website Design

## Goal
Create a modern one-page website for Systems & Services Engineers (Pty) Ltd that converts prospective clients by quickly communicating credibility, services, project experience, and a clear contact path.

## Audience
Prospective clients seeking engineering consultancy, project management, infrastructure, facilities management, power systems, and building services support in Botswana and the region.

## Content Source
The source document is `C:\Users\TOV tech\Downloads\SYSTEMS PROFILE 2026R4.docx`. Content will be adapted for web clarity rather than copied verbatim.

## Site Structure
- Header with brand, section links, and contact action.
- Hero with company name, core positioning, trust indicators, and an engineering/infrastructure visual.
- Credibility band highlighting 28+ years, Botswana citizen ownership, Gaborone base, PPADB registration, and major project delivery.
- About section summarizing experience, office capacity, and professional depth.
- Services section grouped by client need.
- Project portfolio grouped by sector with selected values from the profile.
- Values section for professionalism, accountability, Botho, ethical conduct, and sustainability.
- Final contact call to action.

## Visual Direction
The site should feel like a serious engineering consultancy with a modern, distinctive presentation. Use a restrained palette of white, charcoal, deep teal, signal green, and cool technical blue. Avoid generic card-grid templates by using strong typographic rhythm, technical line motifs, asymmetrical section layouts, and a focused hero image.

## Implementation
Build as a static website in the blank workspace with `index.html`, `styles.css`, `script.js`, and project assets under `assets/`. Include responsive behavior, accessible navigation, semantic sections, and light interaction.

## Acceptance Criteria
- The first viewport clearly identifies Systems & Services Engineers and what the firm does.
- Prospective clients can understand services and project credibility within one scroll.
- The page works on desktop and mobile.
- The page uses at least one real visual asset.
- No placeholder content remains.
